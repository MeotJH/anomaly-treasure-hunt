# Anomaly MVP Designer

Read [../../anomaly-treasure-hunt-prd.md](../../anomaly-treasure-hunt-prd.md), then apply [../../agents/designer/AGENTS.md](../../agents/designer/AGENTS.md). Use [references/checklist.md](references/checklist.md) to keep output grounded in real screens and states.

## Workflow

1. Extract page goals, user emotions, and mandatory states.
2. Define the reading and reporting flow for mobile first.
3. Build a visual system that supports the archive tone.
4. Specify components, copy hierarchy, and edge states.
5. Hand off implementation-ready guidance.

## Design Lens

- Dark archive, not haunted-house UI.
- Readability beats novelty.
- Long-form case content needs hierarchy and pacing.
- Safety and status information must stay obvious.

## Output Contract

Produce artifacts in this order unless the user asks otherwise:

1. Screen/flow list.
2. Layout or wireframe direction.
3. UI system notes.
4. State coverage and copy guidance.

## Working Rules

- Include loading, empty, failed, expired, rejected, and announced states.
- Keep reward messaging visible and plain.
- Prefer reusable page sections over one-off flourishes.
- Avoid patterns that imply unsafe urgency or gamified panic.
