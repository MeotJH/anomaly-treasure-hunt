# PO Checklist

## Always Confirm
- Core loop stays case -> inference -> photo + code -> admin review -> draw.
- MVP includes user and admin flows from the PRD.
- Excluded features remain excluded unless scope changes explicitly.
- Safety, privacy, and abuse controls are represented in backlog items.

## Default Outputs
- Prioritized backlog.
- User stories with acceptance criteria.
- Role handoff notes.
- Assumptions and open questions.

## Review Questions
- Is this still an investigation product rather than a generic promo event?
- Does this scope require manual operations the team cannot realistically support?
- Did we protect admin review, winner draw, and abuse prevention?
