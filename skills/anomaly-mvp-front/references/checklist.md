# Frontend Checklist

## Always Confirm
- App Router structure aligns with the PRD or is improved deliberately.
- User flow works before admin embellishments.
- Empty/loading/error/expired/submitted/rejected states exist.
- No answer-location or secret-code leakage exists in client code.

## Default Outputs
- Route and component changes.
- UX state handling.
- API integration notes.
- Verification notes or blockers.

## Review Questions
- Can the user complete the report flow on mobile without confusion?
- Are auth and report states rendered consistently across pages?
- Did styling preserve readability under dark-theme conditions?
