# DevOps Checklist

## Always Confirm
- Env vars, auth callbacks, buckets, and migration steps are documented.
- Preview and production separation is explicit.
- Secrets and uploaded photos are treated as sensitive.
- Turnstile, logs, and failure visibility are considered.

## Default Outputs
- Config matrix.
- Setup/runbook notes.
- Storage/auth/deployment notes.
- Operational risk list.

## Review Questions
- Can another developer stand up the stack from documentation alone?
- Are there manual steps that should be scripted or at least ordered clearly?
- Do we have enough logging to debug report submission and review failures?
